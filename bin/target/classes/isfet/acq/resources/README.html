Instructions for Running RestletInstrument Examples

The purpose of this exercise was to create an interactive environment using the Restlet REST API to interact with an instrument.
Much of this code was based on, and expanded from code checked in John Krupka for his work using qpid and scxml

The following technologies are used:

  • Restlet
  • Apache QPID
  • Apache SCXML

Steps to run:

  1. Download QPID and install locally
  2. Start the QPID server from ${QPID_HOME}\bin
  3. Start TestServer in isfet.acq package by running as Java Application
  4. Open up restlet_instrument.html in a browser and starting clicking!

Watch any logs for errors. The QPID log is located in ${QPID_HOME}/work (but you must create this directory first!)
Watch the console for error messages - you should see JMS messages being sent back and forth to the server.

The possible order for setting state events is based on the insDm.scxml file. When the instrument is in a certain state, only the events defined for that state can be executed.
For example, when in the "Running" state, the only events that can be executed are "Run Aborted" or "ICRunCompleted".
